Apuntadores de GIT

HEAD

Apuntador Movil que indica donde se encuentra actualmente el usuario dentro del historial de confirmaciones. Se utiliza como referencia relativa para la mayoria de los comandos que requieran manipular los commits y por ajemplo algunas de las expresiones correspondientes pueden ser HEAD~3 o HEAD^^^, si necesito apuntar hacia el commit ancestro que se encuentra a 3 nodos del actual.

  • git diff HEAD^^^ compara el working directory con el commit de hace 3 confirmaciones
  • git checkout HEAD~3 mueve el apuntador hacia el commit ancestro de hace 3 confirmaciones
  • git cherry-pick HEAD~2^ utiliza los cambios del commit de hace 3 confirmaciones
  • git reset HEAD^^~1 elimina los commits hasta el ancestro de hace a 3 confirmaciones
  • git revert HEAD~1^^ crea un commit deshaciendo los cambios del de hace 3 confirmaciones

BRANCH

Apuntador dinamico que se corresponde con el ultimo commit de la rama en la que nos encontramos. Se utiliza como referencia para facilitar la navegacion dentro del historial de confirmaciones. Ademas funciona para probar caracteristicas o resolver inconvenientes con respecto al codigo del proyecto, sin comprometer la historia principal. Los cambios realizados en las mismas pueden ser integrados o no.

  • git branch Lista las ramas locales
  • git branch <name> Crea una rama a partir del commit actual
  • git branch -mv <name> Cambia el nombre de la rama actual
  • git branch -d <branch> elimina la rama solamente si fue integrada (segura)
  • git branch -D <branch> elimina la rama sin aunque no este integrada (forzada)
  • git push <remote> <branch> envia los cambios locales a la rama remota
  • git push -d <remote> <branch> elimina la rama del repositorio remoto

TAG

Apuntador estatico que se corresponde con el commit en donde haya sido creado, se utiliza para el versionado del proyecto y facilita la navegacion hacia puntos concretos dentro del historial de confirmaciones. Normalmente se escriben de la siguiente manera: v0.1.2 donde el primer valor corresponde a la version principal, el segundo al numero de compilacion o caracteristica y el ultimo al numero de correccion o revision.

  • git tag Lista las etiquetas existentes
  • git tag <name> asigna una etiqueta al commit actual
  • git tag <new> <old> asigna una nueva etiqueta en el mismo commit que otra
  • git tag -d <tag> elimina la etiqueta seleccionada
  • git push <remote> --tags envia al repositorio remoto las etiquetas locales
  • git push -d <remote> <tag> elimina la etiqueta del repositorio remoto

STASH

Apuntador de zona temporal de cambios que se utiliza para apilar todo trabajo que no amerita un commit (WIP: Work in Progress), pero que debemos almacenar temporalmente para no perder las modificaciones realizadas. Esto se debe a que GIT no permite cambiar entre ramas si existe algun cambio sin confirmar en el working directory. Una vez finalizada la operacion, podemos recuperar el contenido realizando una operacion merge a la inversa.

  • git stash Almacena los cambios del INDEX en la zona temporal de cambios
  • git stash list Lista los cambios almacenados en la zona temporal
  • git stash apply stash@{n} Aplica los cambios de la zona temporal en el working directory
  • git stash drop stash@{n} Elimina la referencia de la zona temporal de cambios
  • git stash pop stash@{n} Aplica los cambios y elimina el stash de la zona temporal
